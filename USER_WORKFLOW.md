# User Workflows — AI Marketing Platform

Полная карта всех пользовательских сценариев в интерфейсе.

---

## 1. Аутентификация

### 1.1 Регистрация
```
/ → /register
```
1. Вводит имя, email, пароль (мин. 8 символов)
2. Нажимает «Зарегистрироваться»
3. При успехе — редирект на `/dashboard`
4. При ошибке — inline-сообщение (email занят, пароль короткий)

### 1.2 Вход
```
/login
```
1. Вводит email + пароль
2. Нажимает «Войти»
3. Успех → `/dashboard`
4. Ошибка → сообщение «Неверный email или пароль»

### 1.3 Выход
- Клик на аватар в TopBar → выпадающее меню → «Выйти»
- Токен отзывается, редирект на `/login`

### 1.4 Автоматический редирект
- Неаутентифицированный пользователь при любом защищённом маршруте → `/login`
- После входа — возврат к исходному URL

## 2. Проекты

### 2.1 Создание проекта
```
/dashboard (кнопка «Новый проект») → /projects/new
```
1. Вводит название (обязательно) и описание (опционально)
2. Нажимает «Создать проект»
3. Проект становится активным, редирект на `/dashboard`

### 2.2 Выбор проекта
```
/dashboard → ProjectSelector
```
Показывается если нет активного проекта:
1. Видит список своих проектов (карточки с названием, описанием, кол-вом задач)
2. Кликает на карточку → проект становится активным → показывается SMMLDashboard

### 2.3 Смена активного проекта
Два способа:
- Sidebar → секция «Проекты» → клик на название проекта
- Dashboard → кнопка «{Имя проекта} — сменить проект»

### 2.4 Создание проекта из Sidebar (быстрый путь)
- Sidebar → секция «Проекты» → ссылка «Все» → `/dashboard` → «Новый проект»

## 3. Дашборд (`/dashboard`)

После выбора проекта:

### Stat-карточки
- **На согласовании** → клик → `/tasks?status=AWAITING_APPROVAL`
- **Ждёт уточнений** → клик → `/tasks?status=AWAITING_CLARIFICATION`
- **В работе** → клик → `/tasks?status=RUNNING`
- **Готово в этом месяце** → клик → `/tasks?status=COMPLETED`

### Требуют ответа
Список задач со статусом AWAITING_APPROVAL / AWAITING_CLARIFICATION / REVISION_REQUESTED.
- Клик «Открыть» → `/tasks?selected={id}`

### Быстрые сценарии
6 кнопок быстрого запуска задачи по типу:
- Новый пост → `/new?type=post`
- Контент-план → `/new?type=content-plan`
- Reels-сценарий → `/new?type=reels`
- Карусель → `/new?type=carousel`
- Серия историй → `/new?type=stories`
- Промпт для изображения → `/new?type=image-prompt`

## 4. Новый запрос (`/new`)

**Предусловие:** заполнен профиль проекта (companyName, description, niche, geography).

### 4.1 Стандартный путь
1. Выбирает тип задачи (чип из 13 вариантов: Авто-определение / контент / стратегия / кампания)
2. Для контентных типов → выбирает платформы (Instagram, TikTok, Telegram, VK, YouTube, LinkedIn, X, Pinterest)
3. Выбирает приоритет (Срочно / Стандарт / Низкий)
4. Вводит описание задачи в MultimodalInput (textarea)
5. Видит TaskQualityScore (появляется после 3+ слов) — оценка качества по 5 осям
6. Нажимает «Отправить агентам»
7. Редирект на `/tasks?selected={id}`

### 4.2 Прикрепление файлов
В MultimodalInput нажимает иконку скрепки:
- Drag & drop или выбор файла (изображения, PDF, DOCX)
- Файлы отображаются как FileChip под textarea

### 4.3 Добавление URL
В MultimodalInput нажимает иконку ссылки:
- Вводит URL → добавляется в список → прикрепляется к тексту задачи как «Ссылки: ...»

### 4.4 Ошибки при отправке
- Нет профиля → «Перед отправкой заполните профиль: компания, описание, ниша, география»
- Задача слишком расплывчата (score < 25) → «Задача слишком расплывчата. Добавьте контекст, платформу и цель»
- Нет активного проекта → «Выберите проект на дашборде»

## 5. Задачи (`/tasks`)

### 5.1 Просмотр списка
- Два вида: **Список** (вертикальный) / **Канбан** (горизонтальные колонки)
- Переключатель List/Kanban в правом верхнем углу фильтров
- Фильтр по статусу: ALL / RUNNING / AWAITING_APPROVAL / AWAITING_CLARIFICATION / COMPLETED / FAILED
- Автообновление каждые 2 сек при наличии QUEUED/PENDING/RUNNING задач

### 5.2 Канбан-вид
Колонки: Входящие (PENDING/QUEUED) | В работе (RUNNING/AWAITING_CLARIFICATION) | На согласовании (AWAITING_APPROVAL/REVISION_REQUESTED) | Готово (COMPLETED/APPROVED) | Отклонено (FAILED/REJECTED)
- Горизонтальный скролл
- Клик на карточку → детальная панель справа

### 5.3 Выбор задачи
Клик на задачу в списке/канбане → правая панель показывает TaskDetail

### 5.4 Удаление задачи
Hover на карточку задачи → иконка корзины → confirm → DELETE запрос

### 5.5 Редактирование задачи
Только для статусов PENDING / REJECTED:
Hover на текст задачи в TaskDetail → кнопка «Изменить» → textarea → «Сохранить» / «Отмена»

---

## 6. Жизненный цикл задачи

### 6.1 Нормальный путь
```
PENDING → QUEUED → RUNNING → AWAITING_APPROVAL → COMPLETED
```

### 6.2 Путь с уточнением
```
PENDING → QUEUED → RUNNING → AWAITING_CLARIFICATION
```
Пользователь в TaskDetail видит блок «Агенту нужно уточнение» с вопросами.
1. Читает вопросы агента (clarificationNote)
2. Вводит ответ в textarea
3. Нажимает «Ответить»
4. Задача возвращается в RUNNING → AWAITING_APPROVAL

### 6.3 Путь с согласованием
Статус AWAITING_APPROVAL → в TaskDetail появляется ApprovalPanel:
1. Видит результат работы агентов (MARKETER / CONTENT_MAKER / EVALUATOR)
2. Выбирает действие:
   - **«Approve»** → задача переходит в COMPLETED
   - **«Request Revision»** → нужно ввести фидбек (мин. 50 символов) → задача в REVISION_REQUESTED → перезапускается

### 6.4 Живой стриминг
Статус RUNNING → в TaskDetail показывается SSE-поток:
- Анимированная синяя точка «Агент работает»
- Сообщения агентов появляются в реальном времени (AgentAvatar + текст)

### 6.5 Завершённая задача
Статус COMPLETED → блок «Результат» с карточками агентов и их выводами (whitespace-pre-wrap)

### 6.6 Ошибка
Статус FAILED → блок «Задача завершилась с ошибкой»

## 7. Профиль проекта (`/project`)

### 7.1 Просмотр
Три аккордеона:
- **Tier 1 — Базовый** (company, description, niche, geography, products, audience, USP)
- **Tier 2 — Расширенный** (TOV, keywords, forbidden words, website URL)
- **Tier 3 — Полный** (competitors, socialLinks, KPI, existingContent)

### 7.2 Редактирование поля
1. Клик на поле → превращается в input/textarea
2. Вводит значение
3. Нажимает ✓ → PATCH сохраняется, поле закрывается
4. Нажимает ✕ → отмена без сохранения

### 7.3 Выбор TOV
Клик на один из 4 чипов: Официальный / Дружеский / Экспертный / Провокационный

### 7.4 Структурированные поля
- **Продукты / Услуги**: `Название: описание | цена` (одна строка = один продукт)
- **Целевая аудитория**: `Сегмент: портрет | боль1, боль2` (одна строка = один сегмент)
- **Конкуренты**: `Название: URL | позиционирование`
- **Соцсети**: `instagram: url` (одна строка = одна сеть)
- **KPI**: `cac: 500` (числовые поля)

### 7.5 TierProgress (Sidebar)
Показывает % заполнения профиля. Мотивирует заполнить до 100%.

## 8. База знаний (`/project/knowledge`)

### 8.1 Добавление текста
Tab «Текст»:
1. Выбирает категорию (FRAMEWORK / CASE / TEMPLATE / SEO / PLATFORM_SPEC / BRAND_GUIDE)
2. Вводит заголовок (опционально) и текст
3. Нажимает «Добавить»
4. Элемент появляется в списке снизу

### 8.2 Загрузка файла
Tab «Файл»:
1. Drag & drop или выбор файла (PDF, DOCX, TXT, изображения)
2. Выбирает категорию
3. Нажимает «Загрузить»
4. Файл парсится, текст извлекается и сохраняется как knowledge item

### 8.3 Редактирование элемента
В списке: клик иконки карандаша → textarea → ✓ сохранить / ✕ отмена

### 8.4 Удаление элемента
Клик иконки корзины → DELETE запрос → элемент исчезает из списка

### 8.5 Семантический поиск
Поле поиска + кнопка «Найти»:
- Векторный поиск по содержимому базы знаний
- Результаты заменяют общий список
- Очистка поиска → возврат к полному списку

## 9. Настройки (`/settings`)

### 9.1 Tab «Модель AI»
1. Выбирает провайдера (Claude / ChatGPT / Gemini / DeepSeek)
2. Вводит API-ключ
3. (Опционально) изменяет API URL
4. Нажимает «Сохранить» → все новые задачи используют выбранный провайдер
5. Статус показывает: ключ уже сохранён (скрыт) или не задан

### 9.2 Tab «Команда»
- Список участников с ролями (OWNER / MEMBER / VIEWER)
- Клик «Удалить» рядом с участником → DELETE запрос
  - Нельзя удалить последнего OWNER

### 9.3 Tab «Проект»
- Изменить название и описание проекта → PATCH запрос
- Удаление проекта: ввести название для подтверждения → «Удалить» → DELETE → редирект на `/dashboard`

---

## 10. Календарь (`/calendar`)

1. Открывает месячный грид (текущий месяц)
2. Ячейки с завершёнными задачами подсвечены: число + синяя точка
3. Навигация: «Предыдущий» / «Следующий» — переход по месяцам
4. Клик на ячейку дня → правая панель «Задачи за [дата]»:
   - Список завершённых задач этого дня (StatusBadge + input preview)
   - Клик на задачу → `/tasks?selected={id}`

---

## 11. Библиотека (`/library`)

1. Сетка 3 колонки: ArtifactCard для каждой завершённой задачи
2. Каждая карточка:
   - AgentAvatar + тип агента
   - Заголовок задачи (2 строки)
   - Preview вывода агента (120 символов)
   - Дата завершения
3. Клик «Копировать» → вывод агента в clipboard (кнопка меняется на «Скопировано» на 2 сек)
4. Клик стрелки → `/tasks?selected={id}`

---

## 12. Состояния без проекта

Экраны Tasks / Project / Knowledge / Settings / Calendar / Library при отсутствии активного проекта:
- Показывают заглушку с текстом «Проект не выбран»
- Ссылка «Выбрать на дашборде» / редирект на `/dashboard`

## Карта переходов

```
/login ──────────────────────────────────────────→ /dashboard
/register ───────────────────────────────────────→ /dashboard
/dashboard (нет проекта) ──→ /projects/new ──────→ /dashboard (активный проект)
/dashboard ──→ /new ──────────────────────────────→ /tasks?selected={id}
/dashboard (быстрый) ──→ /new?type={type} ────────→ /tasks?selected={id}
/tasks ──→ TaskDetail ──→ ApprovalPanel ──────────→ COMPLETED / REVISION_REQUESTED
/tasks ──→ TaskDetail ──→ ClarificationForm ──────→ RUNNING → AWAITING_APPROVAL
/project ──→ edit field ─────────────────────────→ save (PATCH profile)
/project/knowledge ──→ add/edit/delete/search ───→ (in-place)
/settings ──→ model tab ─────────────────────────→ save (PUT model-config)
/settings ──→ team tab ──→ remove member ─────────→ (in-place)
/settings ──→ project tab ──→ delete ────────────→ /dashboard
/calendar ──→ click day ─────────────────────────→ side panel → /tasks?selected={id}
/library ──→ copy / arrow ───────────────────────→ clipboard / /tasks?selected={id}
```
